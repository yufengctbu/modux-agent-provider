# 意图理解功能设计方案

## 一、背景与动机

### 1.1 当前架构

```
用户输入 → Handler（采集上下文）→ Loop（LLM 多轮调用 + 工具执行）→ 流式输出
```

**现状**：所有用户请求无差别地进入同一个 Agent Loop，由 LLM 自行决定调用哪些工具、如何规划步骤。项目目前没有任何本地意图理解/分类/路由模块。

### 1.2 为什么需要意图理解

| 场景                      | 无意图理解（现状）                          | 有意图理解                                 |
| ------------------------- | ------------------------------------------- | ------------------------------------------ |
| 用户问 "hello"            | 完整走 Loop + 构建 4 层 Prompt → 浪费 token | 识别为闲聊，快速响应，跳过工具列表注入     |
| 用户说 "重构 src/"        | 使用默认 LLM，可能 DeepSeek 吃力            | 识别为复杂重构，切换更强模型或启用更多工具 |
| 用户问 "这个函数什么意思" | 注入全部 12 个工具定义 → 浪费 2000+ token   | 只注入 read_file / lsp_info 等阅读类工具   |
| 用户说 "rm -rf /"         | Loop 正常执行 → 危险                        | 识别为高危操作，强制二次确认或直接拦截     |
| 用户问 "帮我装个 npm 包"  | 走通用 System Prompt                        | 注入前端开发专用提示词，更精准             |

### 1.3 核心价值

- **降低 token 消耗**：按意图裁剪工具列表和 Prompt
- **提升响应质量**：按意图选择最优 LLM / 提示词策略
- **增强安全性**：高危意图提前拦截或升级确认
- **改善用户体验**：简单意图走快速通道，减少等待

## 二、意图分类体系

### 2.1 意图大类（一级分类）

| 意图类别        | 标识        | 说明              | 典型用户输入                                     |
| --------------- | ----------- | ----------------- | ------------------------------------------------ |
| **闲聊 / 问候** | `chat`      | 非编码对话        | "hello", "你好", "你是谁"                        |
| **代码问答**    | `qa`        | 解释/理解代码     | "这个函数是干什么的", "解释一下这段逻辑"         |
| **代码生成**    | `generate`  | 创建新文件/组件   | "创建一个 React 组件", "写一个排序函数"          |
| **代码编辑**    | `edit`      | 修改现有代码      | "把 utils.ts 的重命名", "修复这个 bug"           |
| **代码审查**    | `review`    | 审查/分析代码质量 | "review 一下这个 PR", "检查代码问题"             |
| **调试排错**    | `debug`     | 排查错误/分析日志 | "这个报错怎么回事", "帮忙看看这个 crash"         |
| **重构**        | `refactor`  | 大范围代码重构    | "把这个模块拆成两个", "把 class 改成 functional" |
| **项目操作**    | `project`   | 项目级操作        | "初始化项目", "添加依赖", "配置 ESLint"          |
| **搜索查询**    | `search`    | 信息检索          | "React 19 有什么新特性", "搜一下这个报错"        |
| **命令执行**    | `command`   | 运行命令/脚本     | "跑一下测试", "执行 build"                       |
| **危险操作**    | `dangerous` | 删除/强制推送等   | "删除这个文件", "reset --hard"                   |
| **其他**        | `other`     | 未分类            | 兜底类别                                         |

### 2.2 附加维度（可组合使用）

| 维度          | 值                                            | 用途                               |
| ------------- | --------------------------------------------- | ---------------------------------- |
| **复杂度**    | `simple` / `medium` / `complex`               | 决定是否启用完整工具列表、最大轮次 |
| **紧急度**    | `low` / `normal` / `urgent`                   | 调整 thinking 模式、并发策略       |
| **作用域**    | `single_file` / `multi_file` / `project_wide` | 决定上下文采集范围                 |
| **语言/框架** | `typescript` / `python` / `react` / ...       | 注入领域专用 Prompt                |

## 三、架构设计

### 3.1 整体流程

```
用户输入
  │
  ▼
┌──────────────────────┐
│   IntentClassifier   │  ← 新增模块：意图分类器
│   (classify.ts)      │
└────────┬─────────────┘
         │ 输出: IntentResult { category, complexity, scope, ... }
         ▼
┌──────────────────────┐
│   IntentRouter       │  ← 新增模块：意图路由器
│   (router.ts)        │
└────────┬─────────────┘
         │ 输出: RoutingDecision {
         │   adapter?,       // 可选：指定 LLM 适配器
         │   tools?,         // 可选：工具白名单
         │   systemPrompt?,  // 可选：追加的 System Prompt
         │   maxRounds?,     // 可选：调整最大轮次
         │   skipLoop?,      // 可选：跳过 Loop 直接回复
         │   requireConfirm?,// 可选：需要用户二次确认
         │ }
         ▼
   ┌─────┴─────┐
   │ Handler / │  ← 现有模块，按 RoutingDecision 调整行为
   │ Loop      │
   └───────────┘
```

### 3.2 模块职责

| 模块               | 文件位置                 | 职责                                        |
| ------------------ | ------------------------ | ------------------------------------------- |
| `IntentClassifier` | `src/intent/classify.ts` | 对用户输入进行意图分类，返回 `IntentResult` |
| `IntentRouter`     | `src/intent/router.ts`   | 根据 `IntentResult` 生成 `RoutingDecision`  |
| `IntentConfig`     | `src/intent/config.ts`   | 意图 → 路由策略的映射配置                   |
| `IntentTypes`      | `src/intent/types.ts`    | 类型定义                                    |

### 3.3 两种分类策略（可配置切换）

#### 策略 A：基于规则的分类器（Rule-Based Classifier）

- **优**：零延迟、零 token 消耗、可离线运行
- **劣**：准确率有限、无法理解复杂语义
- **实现**：关键词匹配 + 正则 + 简单启发式规则

```typescript
// 伪代码示例
function classifyByRule(prompt: string): IntentResult {
  if (/^(hi|hello|hey|你好|嗨)/i.test(prompt.trim()))
    return { category: 'chat', complexity: 'simple' }
  if (/(解释|什么意思|这段|这个函数|what does)/i.test(prompt))
    return { category: 'qa', complexity: 'simple' }
  if (/(创建|新建|写一个|create|generate|scaffold)/i.test(prompt))
    return { category: 'generate', complexity: 'medium' }
  if (/(修改|改成|改一下|修复|fix|change|update|edit)/i.test(prompt))
    return { category: 'edit', complexity: 'medium' }
  if (/(review|审查|检查|代码质量|code review)/i.test(prompt))
    return { category: 'review', complexity: 'medium' }
  if (/(报错|错误|error|bug|crash|不工作|调试|debug)/i.test(prompt))
    return { category: 'debug', complexity: 'medium' }
  if (/(重构|拆分|重写|refactor|rewrite|restructure)/i.test(prompt))
    return { category: 'refactor', complexity: 'complex' }
  if (/(删除|rm\s|reset --hard|force push|覆盖)/i.test(prompt))
    return { category: 'dangerous', complexity: 'medium' }
  if (/(搜索|查一下|搜|search|google|查文档)/i.test(prompt))
    return { category: 'search', complexity: 'simple' }
  // ...更多规则
  return { category: 'other', complexity: 'medium' }
}
```

#### 策略 B：基于 LLM 的分类器（LLM-Based Classifier）

- **优**：准确率高、能理解复杂语义、可多维度同时分类
- **劣**：有延迟、消耗 token
- **实现**：用轻量/快速的 LLM（或同一 LLM 但限制 max_tokens=50）做一次分类调用

```typescript
// 伪代码示例
// ⚠️ 安全要点：用户输入必须用明确分隔标签包裹，防止 Prompt Injection 操纵分类结果
const CLASSIFY_PROMPT = `Classify the following user message into one of these categories:
chat, qa, generate, edit, review, debug, refactor, project, search, command, dangerous, other.

Also assess:
- complexity: simple / medium / complex
- scope: single_file / multi_file / project_wide
- language: TypeScript / JavaScript / Python / Go / Rust / unknown

Return ONLY valid JSON: {"category":"...","complexity":"...","scope":"...","language":"..."}

User message:
<user_input>
${prompt}
</user_input>`
```

#### 推荐策略：混合模式（Hybrid）

1. **先走规则分类**（零延迟），命中高置信度规则时直接返回
2. **低置信度时走 LLM 分类**（兜底），保证准确率
3. 结果缓存：同一轮对话内，第一次分类结果缓存到 `vscode.ChatContext`，后续轮次复用

> **实现提醒**：`classify.ts` 需导出顶层协调函数 `classifyIntent(prompt): Promise<IntentResult>`，内部编排"规则 → LLM 兜底 → 缓存"逻辑。`handler.ts` 中 `await classifyIntent(...)` 调用的正是此函数，不能只导出 `classifyByRule`。

### 4.1 类型定义 (`src/intent/types.ts`)

```typescript
/** 意图大类 */
export type IntentCategory =
  | 'chat' // 闲聊/问候
  | 'qa' // 代码问答
  | 'generate' // 代码生成
  | 'edit' // 代码编辑
  | 'review' // 代码审查
  | 'debug' // 调试排错
  | 'refactor' // 重构
  | 'project' // 项目操作
  | 'search' // 搜索查询
  | 'command' // 命令执行
  | 'dangerous' // 危险操作
  | 'other' // 未分类

/** 分类置信度 */
export type Confidence = 'high' | 'medium' | 'low'

/** 意图分类结果 */
export interface IntentResult {
  /** 主类别 */
  category: IntentCategory
  /** 分类置信度 */
  confidence: Confidence
  /** 任务复杂度 */
  complexity: 'simple' | 'medium' | 'complex'
  /** 作用域 */
  scope: 'single_file' | 'multi_file' | 'project_wide'
  /** 检测到的编程语言/框架 */
  language: string
  /** 分类来源：rule | llm | cached */
  source: 'rule' | 'llm' | 'cached'
}

/** 路由决策 */
export interface RoutingDecision {
  /** 覆盖默认 LLM 适配器类型（可选） */
  adapterType?: string
  /** 工具白名单：只启用这些工具；undefined 表示全部启用 */
  toolWhitelist?: string[]
  /** 追加到 System Prompt 的内容 */
  appendSystemPrompt?: string
  /** 覆盖最大循环轮次（可选） */
  maxRounds?: number
  /** 是否跳过 Agent Loop 直接返回（简单闲聊） */
  skipLoop?: boolean
  /** 直接回复内容（skipLoop=true 时使用） */
  directReply?: string
  /** 是否需要用户二次确认 */
  requireConfirm?: boolean
  /** 确认提示语 */
  confirmMessage?: string
}
```

### 4.2 意图分类器 (`src/intent/classify.ts`)

```typescript
import type { IntentResult, IntentCategory, Confidence } from './types'

// ── 规则定义 ────────────────────────────────────────────────

interface ClassifyRule {
  pattern: RegExp
  category: IntentCategory
  confidence: Confidence
  complexity: IntentResult['complexity']
  scope: IntentResult['scope']
}

const HIGH_CONFIDENCE_RULES: ClassifyRule[] = [
  // 闲聊
  {
    pattern: /^(hi|hello|hey|你好|嗨|早上好|晚上好)\b/i,
    category: 'chat',
    confidence: 'high',
    complexity: 'simple',
    scope: 'single_file',
  },
  {
    pattern: /^(你是|你是谁|你叫什么|what are you)/i,
    category: 'chat',
    confidence: 'high',
    complexity: 'simple',
    scope: 'single_file',
  },

  // 危险操作
  {
    pattern: /\brm\s+-rf\b/i,
    category: 'dangerous',
    confidence: 'high',
    complexity: 'medium',
    scope: 'project_wide',
  },
  {
    pattern: /\bgit\s+reset\s+--hard\b/i,
    category: 'dangerous',
    confidence: 'high',
    complexity: 'medium',
    scope: 'project_wide',
  },
  {
    pattern: /\bgit\s+push\s+--force\b/i,
    category: 'dangerous',
    confidence: 'high',
    complexity: 'medium',
    scope: 'project_wide',
  },
  {
    pattern: /\bDROP\s+(TABLE|DATABASE)\b/i,
    category: 'dangerous',
    confidence: 'high',
    complexity: 'medium',
    scope: 'project_wide',
  },

  // 调试
  {
    pattern: /(报错|错误信息|error|exception|stack trace|crash|崩溃)/i,
    category: 'debug',
    confidence: 'high',
    complexity: 'medium',
    scope: 'single_file',
  },

  // 重构
  {
    pattern: /(重构|大改|重写|refactor|rewrite\s+(the|this|all))\b/i,
    category: 'refactor',
    confidence: 'high',
    complexity: 'complex',
    scope: 'multi_file',
  },

  // 项目操作
  {
    pattern: /(初始化|init|scaffold|创建项目|create\s+(react|next|vue).*app)/i,
    category: 'project',
    confidence: 'high',
    complexity: 'complex',
    scope: 'project_wide',
  },
]

const MEDIUM_CONFIDENCE_RULES: ClassifyRule[] = [
  {
    pattern: /(解释|什么意思|这段|这个(函数|方法|类|变量)|what\s+does)/i,
    category: 'qa',
    confidence: 'medium',
    complexity: 'simple',
    scope: 'single_file',
  },
  {
    pattern: /(创建|新建|写一个|添加|generate|create\s+a|scaffold)/i,
    category: 'generate',
    confidence: 'medium',
    complexity: 'medium',
    scope: 'single_file',
  },
  {
    pattern: /(修改|改成|改一下|修复|fix|change|update.*code|edit)/i,
    category: 'edit',
    confidence: 'medium',
    complexity: 'medium',
    scope: 'single_file',
  },
  {
    pattern: /(review|审查|检查|代码质量|code\s+review|审计)/i,
    category: 'review',
    confidence: 'medium',
    complexity: 'medium',
    scope: 'multi_file',
  },
  {
    pattern: /(搜索|查一下|搜|search|google|查文档|怎么.*用)/i,
    category: 'search',
    confidence: 'medium',
    complexity: 'simple',
    scope: 'single_file',
  },
  {
    pattern: /(运行|执行|跑一下|run\s+|execute|npm\s+(run|test|build)|yarn\s+|pnpm\s+)/i,
    category: 'command',
    confidence: 'medium',
    complexity: 'simple',
    scope: 'single_file',
  },
]

// ── 分类函数 ────────────────────────────────────────────────

export function classifyByRule(prompt: string): IntentResult | null {
  const trimmed = prompt.trim()

  // 先匹配高置信度规则
  for (const rule of HIGH_CONFIDENCE_RULES) {
    if (rule.pattern.test(trimmed)) {
      return {
        category: rule.category,
        confidence: rule.confidence,
        complexity: rule.complexity,
        scope: rule.scope,
        language: detectLanguage(trimmed),
        source: 'rule',
      }
    }
  }

  // 再匹配中等置信度规则
  for (const rule of MEDIUM_CONFIDENCE_RULES) {
    if (rule.pattern.test(trimmed)) {
      return {
        category: rule.category,
        confidence: rule.confidence,
        complexity: rule.complexity,
        scope: rule.scope,
        language: detectLanguage(trimmed),
        source: 'rule',
      }
    }
  }

  // 无匹配，返回 null 表示需要 LLM 分类
  return null
}

/** 简单语言检测（从 prompt 和上下文推断） */
function detectLanguage(prompt: string): string {
  // tsx? 匹配 .ts/.tsx，jsx? 匹配 .js/.jsx，必须分开判断，避免 JS 被误归为 TypeScript
  if (/\.tsx?\b/i.test(prompt)) return 'TypeScript'
  if (/\.jsx?\b/i.test(prompt)) return 'JavaScript'
  if (/\.py\b/i.test(prompt)) return 'Python'
  if (/\.go\b/i.test(prompt)) return 'Go'
  if (/\.rs\b/i.test(prompt)) return 'Rust'
  if (/\b(React|Next\.js|Vue|Angular|Svelte)\b/i.test(prompt)) return 'TypeScript'
  return 'unknown'
}
```

### 4.3 意图路由器 (`src/intent/router.ts`)

```typescript
import type { IntentResult, RoutingDecision } from './types'
import { intentRoutingConfig } from './config'

export function routeByIntent(intent: IntentResult): RoutingDecision {
  // intentRoutingConfig 改为 Partial<Record<...>>，未配置的类别走默认全能力流程
  const route = intentRoutingConfig[intent.category]
  if (!route) return {}

  return {
    adapterType: route.adapterType,
    toolWhitelist: route.toolWhitelist,
    appendSystemPrompt: route.appendSystemPrompt,
    // ⚠️ 运算符优先级：?? 低于 ===，必须显式括号包住三元表达式
    maxRounds: route.maxRounds ?? (intent.complexity === 'simple' ? 3 : undefined),
    skipLoop: route.skipLoop,
    directReply: route.directReply,
    requireConfirm: route.requireConfirm,
    confirmMessage: route.confirmMessage,
  }
}
```

### 4.5 集成到现有流程

#### 修改 `src/chat/handler.ts`

```typescript
export async function handleChatRequest(
  request: vscode.ChatRequest,
  context: vscode.ChatContext,
  stream: vscode.ChatResponseStream,
  token: vscode.CancellationToken,

  log(`[Handler] 收到消息：${request.prompt}`)

  // ── 新增：意图分类与路由 ────────────────────────────
  const intent = await classifyIntent(request.prompt)
  const routing = routeByIntent(intent)
  log(`[Handler] 意图=${intent.category} 置信度=${intent.confidence} 来源=${intent.source}`)

  // 危险操作：强制拦截，必须 return，不得穿透到 runAgentLoop
  if (routing.requireConfirm) {
    stream.markdown(routing.confirmMessage ?? '⚠️ 检测到潜在危险操作，请确认。')
    // 实现选项 A（推荐）：将 requireConfirm 传入 loop，首轮强制调用 ask_user 工具发起确认
    // 实现选项 B：暂时直接 return 拒绝执行，提示用户明确确认后重新提交
    return
  }

  // 纯闲聊快速通道
  if (routing.skipLoop) {
    if (routing.directReply) {
      stream.markdown(routing.directReply)
      return
    }
    // directReply 为空时走单轮无工具 QuickChat，而非完整 Loop
    const wsCtx = await getWorkspaceContext()
    await runQuickChat(request.prompt, stream, token, routing.appendSystemPrompt)
    log('[Handler] QuickChat 完成')
    return
  }

  // ── 原有流程（传路由决策） ──────────────────────────
  const wsCtx = await getWorkspaceContext()
  await runAgentLoop(request.prompt, context, stream, token, wsCtx, undefined, routing)
  log('[Handler] 请求处理完成')
}
}
```

## 八、备选方案（如果不想大改）

### 极简方案：仅做危险操作检测

只新增一个 `dangerous` 意图检测，在 handler 中做关键词正则匹配，命中时强制确认。不改变原有流程，改动最小。

```typescript
// 只需在 handler.ts 添加 10 行代码
const DANGEROUS_PATTERNS = [/rm\s+-rf/, /git\s+reset\s+--hard/, /DROP\s+(TABLE|DATABASE)/i]
if (DANGEROUS_PATTERNS.some((p) => p.test(request.prompt))) {
  stream.markdown('⚠️ 检测到潜在危险操作，请确认后再执行。')
  return
}
```

### 中等方案：仅做工具裁剪

基于简单规则识别意图大类，只裁剪工具列表（`toolWhitelist`），不改 Prompt、不改适配器。改动局限在 `loop.ts` 的工具过滤逻辑。
